Authentification

Ajout de la possibilité de s’authentifier sur les serveurs de l’UTT

utt_auth.cas(service: str, identifiant: str = '', mdp: str = '', session_web: requests.sessions.Session = <requests.sessions.Session object>) → requests.sessions.Session

Permet d’initier une session web sur un service via le CAS de l’UTT, soit de manière manuelle, soit de manière automatique en fournissant id, mdp, et session web si besoin

Date

Aout 2019

Author

Ivann LARUELLE

Paramètres

  • service (str) – L’url du service auquel on veut accéder via le CAS

  • identifiant – L’identifiant de l’user en mode auto

  • mdp (str) – Le mdp de l’user en mode auto

  • session_web (requests.sessions.Session) – On peut ajouter la connexion CAS à une session web déjà existante

Renvoie

Une session avec laquelle on peut effectuer des requêtes, ou -1 en mode auto si échec

Type renvoyé

requests.sessions.Session

Lève

Exception – Si l’id/mdp ne sont pas corrects ou connexion impossible au CAS

utt_auth.smtp(identifiant: str = '', mdp: str = '') → smtplib.SMTP_SSL

Permet d’initier une session SMTP sécurisée sur le serveur mail de l’UTT, soit de manière manuelle, soit de manière automatique en fournissant id et mdp

Date

Aout 2020

Author

Ivann LARUELLE

Paramètres

  • identifiant (str) – L’identifiant de l’user en mode auto

  • mdp (str) – Le mdp de l’user en mode auto

Renvoie

Une session avec laquelle on peut effectuer des requêtes, ou -1 en mode auto si échec

Type renvoyé

smtplib.SMTP_SSL

Lève

  • smtplib.SMTPAuthenticationError – Si le couple id/mdp est invalide

  • Exception – si le serveur est injoignable
